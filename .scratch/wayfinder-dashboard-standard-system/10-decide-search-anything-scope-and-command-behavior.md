**Blocked by:** [Decide dashboard IA, route map, and role-based navigation](04-decide-dashboard-ia-route-map-and-role-based-navigation.md), [Define dashboard API contracts and server-action boundary](06-define-dashboard-api-contracts-and-server-action-boundary.md)

**Status:** open

## Question

What should EwaTrade's Midday-style search-anything feature search, open, and command in the dashboard?

## Context

The user explicitly wants the search-anything feature exactly in the spirit of Midday. Search should respect tenant, store, role, and permission boundaries. It may need to search businesses, products, inventory units, customers, staff, sales, links, reports, settings, and actions such as create product or invite staff.

## Resolution Target

Decide:

- Search keyboard shortcut and entry points.
- Search result types for the first implementation.
- Command actions versus pure navigation results.
- Permission and tenant filtering rules.
- Empty, loading, and error states.
- Whether search uses a dedicated tRPC endpoint, local command registry, or a hybrid.
