## Question

What schema and API shape should connect inbound costing to the existing stock ledger without duplicating `StockDelivery`, `StockDeliveryLine`, `InventoryMovement`, product units, or price history?

Research and decide at least:

- Whether to add new inbound header, line, cost-line, allocation, and approval tables.
- Whether `StockDelivery` should remain the stock posting/audit object while inbound records remain the costing/planning object.
- Whether production batches need a separate source type or can share inbound records with a mode.
- How to link inbound lines to products, variants, inventory items, stock delivery lines, and inventory movements.
- How to model landed cost snapshots, cost layers, and margin-report inputs.
- How idempotency/external ids should work for offline or poor-network submissions.
- Which tRPC procedures should exist for create/update/approve/post/list/detail.
- What migration and rollout path keeps existing stock intake working during transition.

The answer should include a recommended entity relationship sketch and migration sequence, but not implementation code.
